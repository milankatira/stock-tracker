<objective>
Ship the public marketing landing page at `/` — a single static, fully-responsive, SEO-indexed page that pitches FinSight AI's value prop, shows a compliance-safe sample report, lists features + personas + how-it-works + pricing teaser + compliance strip + FAQ, and drives sign-ups via a CTA to the existing `/signup` route (created by Phase 1).

Purpose:
- Convert organic + paid traffic into account sign-ups (the funnel entry point).
- Establish the compliance-safe brand voice in the most copy-heavy public surface.
- Provide a search-indexable home that complements the Phase 8 per-stock/per-fund SEO pages.

Output:
- Static Server Component at `apps/web/src/app/page.tsx` (`force-static`), composed of ~10 section components under `apps/web/src/components/landing/`.
- `@vercel/analytics` wired in the root layout (cookieless, no DPDP consent burden).
- OG / Twitter / canonical metadata + JSON-LD Organization + FAQPage.
- Vitest copy-compliance test that fails the build if any forbidden marketing verb appears in rendered DOM.
- GitHub Actions workflow that gates merges on Lighthouse mobile >= 95 + axe-core zero violations.

LAND-01 and LAND-02 are both satisfied by this single plan.
</objective>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser → CDN | Public, unauthenticated. Static HTML served from edge. No PII collected. |
| CTA → /signup | Boundary into authenticated auth flow (Phase 1). This plan only emits a static `<Link>` — Phase 1 owns the validated boundary. |
| TS source → rendered DOM | Marketing copy is the highest-risk surface for SEBI compliance creep (BUY/SELL framing leaking in). |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-09-01 | Tampering | XSS via copy / CMS injection | mitigate | All copy is statically authored in TS source (`data.ts`). No CMS. The two `dangerouslySetInnerHTML` JSON-LD blocks consume `JSON.stringify(...)` of typed objects — safe. CSP `default-src 'self'` set in `next.config.ts`. |
| T-09-02 | Tampering | Clickjacking — page embedded in malicious iframe | mitigate | `X-Frame-Options: DENY` + CSP `frame-ancestors 'none'` via `next.config.ts` headers (Task 2). |
| T-09-03 | Tampering | Third-party analytics script injection | mitigate | First-party `@vercel/analytics` only (no third-party origins). CSP `script-src 'self' 'unsafe-inline' va.vercel-scripts.com` allowlist. Any future analytics requires a perf + CSP review. |
| T-09-04 | Spoofing | Open redirect via CTA query param | mitigate | CTA is a literal `<Link href="/signup">` — no query param interpolation, no redirect logic. The forbid-list test would catch a copy slip; the static type system catches an href change. |
| T-09-05 | Repudiation (legal) | SEBI compliance violation — page uses BUY/SELL/recommend verbs | mitigate | Copy-compliance Vitest test (Task 3) scans rendered DOM + JSON-LD against a 16-item forbid-list and fails the build on any hit. Extends COMP-01 verdict enum invariant to the marketing surface. |
| T-09-06 | Information Disclosure | Stack traces / debug info in 500 page | accept | Next.js prod build masks errors by default; the landing page has no runtime branches that can 500 (force-static + no data fetching). Low risk — accept. |
| T-09-07 | DoS | High-volume crawler traffic on `/` | accept | Static HTML at the edge (CDN-cached). Vercel/CDN absorbs load; no origin compute per request. Accept. |
| T-09-08 | Elevation of Privilege | Analytics gains access to PII | mitigate | `@vercel/analytics` is cookieless by default; no PII collected on landing (no form, no auth state). DPDP-friendly. Documented in research as the rationale for choosing it over Plausible/PostHog. |
