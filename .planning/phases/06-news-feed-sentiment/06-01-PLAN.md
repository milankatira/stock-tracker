<objective>
Stand up the news ingestion pipeline: RSS + NewsData.io adapters behind the existing DataIngestion patterns, a `News` Mongoose schema with the Atlas Vector Search index committed alongside it, ticker-mention extraction with the over-attribution fix, deduplication, and a BullMQ `news-poll` repeatable job that fans out to per-source then per-article processors. Embeddings + sentiment are NOT generated in this plan вЂ” articles are persisted with `classificationStatus: 'pending'` and `embedding: undefined`, which Plan 02 then completes.

Purpose: Materialise per-instrument news into MongoDB so the report read path is a pure Redis в†’ Mongo lookup, and lay the storage shape (vector field + filter indexes) Plan 02 will populate. The Atlas index is committed AND boot-asserted in this plan because the schema and the index travel together вЂ” a wrong-dimension index silently corrupts queries (Pitfall 2).

Output: NewsModule wired into AppModule; News collection with unique (source, externalId); Atlas Vector Search index `news_embedding_idx` defined and asserted; news-poll job repeatable every 30 min with per-source fan-out; `GET /stocks/:ticker/news` returning the persisted feed; ticker-tagger unit-tested for the Adani-style over-attribution case; RSS URL boot probe logging fail-loud on any 4xx/5xx.
</objective>
